Wireless Networks Notes

Posted on 2017-08-01

Selected issues of wireless communications

Path loss. $1/r^2$ in free space and $1/r^3$ to $1/r^4$ indoor.

Slow fading (shadowing)

  • caused by obstacles in the transmission line (houses, trees, etc.)
  • is a random attenuation on top of path loss
  • varies in time → changes within seconds

Fast fading (sometimes just referred to as fading)

  • caused by multi-path propagation
  • is a random attenuation on top of path loss and slow fading
  • always varies in frequency → frequency-selective fading if communication channel is larger than the “width of attenuation variation in frequency”
  • Usually also varies in time (time-selective fading), if in combination with multipath propagation transmitter and/or receiver (or some reflectors) are moving!

MAC in wireless networks

Duplex: FDD, TDD(Time Division Duplex)
Multiple Access methods:
SDMA (Space Division Multiple Access): use directed antennas.
FDMA (permanent (e.g., radio broadcast), slow hopping (e.g., GSM), fast hopping (FHSS, Frequency Hopping Spread Spectrum))
TDMA
CDMA (assign a specific code to a transmission channel between a sender and a receiver for a certain amount of time - a large scope of frequency used)

Centralized Medium Access

Idea: have a central station controlling when a node may access the medium.
Not directly feasible for non-trivial wireless network sizes.
But: can be quite useful when network is somehow divided into smaller groups: Clusters (for example, Bluetooth piconets)

CSMA

Carrier Sense Multiple Access
reduce collisions using carrier sensing

  • Listen-before-you-talk
  • A host may transmit only if the channel is sensed as idle

In wired world: Collision Detection (CD). Stop transmitting immediately once collision is detected.
Two types of overlapping transmissions:
simultaneous (time), concurrent (space).

Hidden Terminal Problem: A is hidden from C; C transmits to B while A is also transmitting to B → collision → packet losses.

Exposed Terminal Problem: C is exposed to B; when C wants to transmit to D, it defers unnecessarily while B is transmitting to A → low channel utilization.

Solutions for Hidden Terminals

Busy-tone
Uses two frequency bands:

  • the data band is used to transmit data packets
  • the busy-tone band is used to transmit busy-tone signals

Approach: while receiving a packet a station sends an ongoing signal on the busy-tone band. The carrier sensing is performed on the busy-tone channel.

Virtual carrier sensing

Collision avoidance is based on virtual carrier sense, not on physical carrier sense because PCS gives information about collision at the sender, not at the receiver.

  • “Reserve” the wireless channel before transmitting data
  • Send short control packets for reservation:
    • RTS = Request-to-Send: sender requests the right to send from a receiver
    • CTS = Clear-to-Send: the receiver grants the right to send

IEEE 802.11 DCF

Distributed and centralized MAC components:

  • Distributed Coordination Function (DCF)
  • Point Coordination Function (PCF)

DCF suitable for multi-hop ad hoc networking.
DCF is a Carrier Sense Multiple Access/Collision Avoidance (CSMA/CA) protocol.

  • Physical and virtual carrier sensing (RTS-CTS)
  • Contention window (CW) : backoff chosen uniformly in [0, CW − 1]
  • Exponential backoff after a packet loss
  • Contention window reset to CWmin on a success
  • Uses ACK to achieve reliability

The time spent counting down backoff intervals is a part of MAC overhead.
CW is chosen dynamically depending on collision occurrence.

IEEE 802.x framework
Duplexing: The signals moving between two elements in opposite directions can be separated using time division duplexing (TDD) or frequency division duplexing (FDD). In the case of CSMA, duplexing is not relevant.

FHSS (Frequency Hopping Spread Spectrum)
DSSS (Direct Sequence Spread Spectrum)
RADIUS (Remote Authentication Dial-In User Service)

FCS (Frame Check Sequence)

Protocol: Indicates IEEE 802.11 MAC
Type: 00 (Management frames), 01 (Control frames), 10 (Data frames).
Subtype of frame: Describes type of management, control, or data frame in more detail (e.g. ACK => 1101).
Flags 1: Bit is set if frame is sent to AP, 2: Bit is set if frame is sent from AP, 3: Used in fragmentation, 4: Bit is set if frame is retransmitted, 5: Power management bit (power saving operation), 6: More data bit (power-saving operation), 7: Bit is set if WEP is used, 8: Strict ordering of frames is required.

Address 1: Receiver (wireless station or AP)
Address 2: Sender (wireless station or AP)
Address 3: Ultimate source/destination (router in DS(Distribution System))
Address 4: Only used in Wireless Bridge solutions.
Direction: Wireless station -> AP

WLAN MAC

CSMA/CD rule: Backoff after collision.
CSMA/CA rule: Backoff before collision.
Basic CSMA/CD operation: 1) Wait for free medium. 2) Transmit frame. 3) If collision, stop transmission immediately. 4) Retransmit after random time (backoff)
Basic CSMA/CA operation: 1)Wait for free medium. 2) Wait a random time (backoff). 3) Transmit frame. 4) If collision, the stations do not notice it. 5) Collision => erroneous frame => no ACK returned.

DCF (Distributed Coordination Function) based on CSMA/CA, used for contention services.
2 mechanisms for reserving the channel:
Physical carrier sensing and Virtual carrier sensing using the socalled Network Allocation Vector (NAV).

Physical carrier sensing means that the physical layer (PHY) informs the MAC layer when a frame has been detected.

Virtual carrier sensing means that a NAV value is set in all stations that were able to receive a transmitted frame and were able to read the NAV value in this frame.

SIFS (Short InterFrame Space) = 10 µs (16 µs) / 802.11b (802.11g)
DIFS (DCF InterFrame Space) = 50 µs (34 µs)
Duration field in header: 15 bits contain the NAV value in number of microseconds. The last (sixteenth) bit is zero.

Contention window (CW)
Initial attempt, CW = 16-1 = 15 slots (slot = 9 µs) in 802.11g.
ACK frame structure : with only one addr. (frame sender)

WLAN PHY

802.11b

ISM 2.4G band, channel spacing = 5MHz.
IEEE 802.11b frame structure (PHY layer)

Preamble - synchronizes the transmitter and receiver

  • Sync field - 128-bit field composed entirely of 1s
  • Start Frame Delimiter - allows the receiver to find the start of the frame (e.g. 0000 0101 1100 1111)

Header - has PHY-specific parameters used by the PLCP

  • Signaling field used to identify the transmission rate of the encapsulated MAC frame
  • Service identification field (not used)
  • Length field
  • 16-CRC

802.11a/g

OFDM (Orthogonal Frequency Division Multiplexing)
The information is carried over the radio medium using orthogonal subcarriers. A channel (16.25 MHz wide) is divided into 52 subcarriers (48 subcarriers for data and 4 subcarriers serving as pilot signals).
Subcarriers are modulated using BPSK, QPSK, 16-QAM, or 64-QAM, and coded using convolutional codes (R = 1/2, 2/3, and 3/4), depending on the data rate.

802.11e

ITU-T Recommendation G.114 states that the round-trip delay should be less than 300 ms for telephony.
PER (packet error ratio)
QoS solutions in 802.11 networks

  • The 802.11 standard defines the Point Coordination Function (PCF) for carrying real-time traffic. This solution has not been widely implemented.
  • A number of advanced QoS solutions have been defined in the 802.11e standard (approved in 2005).

PCF operation

B = Beacon frame (sent by AP to indicate start of CFP)
CFP(Contention-Free Period) is reserved for real-time traffic.
CP(Contention Period) normal DCF operation
Undisturbed CFP operation is guaranteed in two ways:
• The NAV value in the beacon signal = length of CFP
• Usage of PIFS within CFP (instead of DIFS), PIFS < DIFS
PCF is based on polling.

Shortcomings:

  • Differentiation between traffic classes is not possible
  • No mechanisms for wireless stations to communicate QoS requirements to the access point
  • The contention free period (CFP) length cannot be dynamically changed according to traffic needs

The 802.11e standard defines a new Hybrid Coordination Function (HCF) that offers two modes of operation:
Enhanced DCF (EDCF) is like DCF, but introduces different priority levels for different services.
HCF Controlled Channel Access (HCCA) is a CSMA/CA-compatible polling-based access method.

EDCF

Channel access is controlled by using four differentiating parameters:

  • Minimum contention window size (CWmin)
  • Maximum contention window size (CWmax)
  • Arbitration Interframe Space (AIFS) = variable DIFS
  • Transmission Opportunity (TXOP) specifies the time (maximum duration) during which a wireless station can transmit a series of frames.
    The IEEE 802.1D(p) standard defines 8 Access Categories (AC) for differentiating users that have different priority requirements.

HCCA

HCCA can poll stations also during the Contention Period (CP).
Stations can communicate their QoS requirements (data rate, delay, packet size…) to the access point.
HCCA supports scheduling of packets based on the QoS requirements.

Voice (speech) coding schemes

Standard PCM (Pulse Code Modulation) produces a fixed bit rate of 64 kbit/s.
G.726 specifies an Adaptive Differential PCM (ADPCM) codec which produces various bit rates (16, 24, 32, or 40 kbit/s).

Approach: distribute FA functionality between multiple FA to realize improved support for local mobility.

FMIPv4

Mobile IPv4 Fast Handovers (FMIPv4) (RFC 4988)
Idea: avoid disconnection during handover by bi-casting traffic to the previous FA and new FA while MH is moving between them.

requires an MH to be able to communicate with two FAs at the same time.
MH sends solicitation to new FA before handover.

Mobile IPv6

No need for a separate FA, all routers perform router advertisement which can be used instead of the special agent advertisement.
MH can signal a sender directly the CoA, sending via HA not needed in this case (automatic path optimization).
Address auto-configuration is inherent in IPv6.

basically a kind of FA for Mobile IPv6 with the different purpose of speeding up HOs and not requiring direct link layer connectivity to MH.
MH uses MAP address (learned from router advertisements) as alternate CoA and MAP receives and tunnels packets to MH’s current address.

Additional features:
MH can register with more than one MAP simultaneously and use each MAP address for a specific group of correspondent hosts.
MAP can keep simultaneous bindings and bi-cast packets to MH on both paths when mobile host does handover.

TCP over wireless

Impact of transmission errors on TCP performance
Every packet loss is interpreted as an indication for congestion resulting in a reduced sending rate!
Now, in wireless network:

  • If number of errors is small, they may be corrected by an error correcting code.
  • Excessive bit errors result in a packet being discarded, possibly before it reaches the transport layer.

Reducing congestion window in response to errors is unnecessary.

If wireless link remains unavailable for extended duration, a window worth of data may be lost. (when a car driving through a tunnel)
Then Timeout results in slow start.

Solution:
Classification #1, based on nature of actions
Hide error losses from the sender

  • if sender is unaware of the packet losses due to errors, it will not reduce cwnd
    Let sender know, or determine, cause of packet loss
  • if sender knows that a packet loss is due to errors, it will not reduce cwnd.

Classification #2, where modifications are needed
At the sender node only
At the receiver node only
At intermediate node(s) only
Combinations of the above

Link-level retransmissions

Best results among different link-layer retransmission strategies are achieved with an immediate acknowledgement within the MAC protocol.
Classification

  1. Hide wireless losses from TCP sender.
  2. Link layer modifications needed at both ends of wireless link.

Split connection approach

End-to-end TCP connection is broken into one connection on the wired part of route and one over wireless part of the route.
Split connection results in independent flow control for the two parts.
If wireless link is not last on route, then more than two TCP connections may be needed.

Example: Indirect TCP (I-TCP)

Principle: Separate transport connection used for fixed and wireless segments. End-to-End connectivity support via a transport layer gateway placed in the base station.
Separation of flow and congestion control of the wireless link from that of the wired part.
specialized protocol (a modified version - TCP) used for wireless part.

Example: Mobile TCP

Low overhead protocol at wireless hosts.
No congestion control over wireless hop.
Preserve TCP’s semantic by using a “Redirector” on the base station part.

Example: M-TCP

Terminate the TCP connection at BS.

  • BS pretends to be MH (MH’s IP functionality moved to BS).
  • BS-MH link can use any arbitrary protocol optimized for wireless link.

TCP-aware link layer

Example: Snoop TCP
Retains local recovery of split connection approach and link level retransmission schemes.
Improves on split connection:

  • end-to-end semantics retained
  • soft state at base station, instead of hard state

Snoop instance (located within a base station) can receive, interpret, filter and generate TCP segments.
Snoop-function:

  • Keeps track of unacknowledged packets (cache) and ACKs.
  • Cares for local retransmission and ACKs for the sender (FH).
  • Filtering of duplicate ACKs - prevents fast retransmit at TCP sender (FH) by dropping the Dup ACKs at BS.

Bluetooth

Bluetooth technology allows of the replacement of many propriety cables that connect one device to another with one universal short-range radio-link.
Bluetooth® Basic Rate/Enhanced Data Rate (BR/EDR)

Piconet

All devices can be acting as a master/slave.
Piconet is established and controlled by master.
Communication between slaves is not possible.
Only 3 bits used for Bluetooth MAC: up to 7 slaves can be connected to master.
Interpiconet communication shall be possible by forwarding nodes (special slaves).

Architecture

Bluetooth is defined as a layer protocol architecture consisting of core protocols, cable replacement protocols, telephony control protocols, and adopted protocols.
Mandatory protocols for all Bluetooth stacks are: LMP, L2CAP and SDP.
Additionally, these protocols are almost universally supported: HCI and RFCOMM.

Bluetooth baseband

Time Division Duplex (TDD): first slot is reserved for master, second slot is occupied by slave.
Hopping is performed on a per-slot basis - 1600 hop/s means one hop every 0.625 ms.

Packet format

3 fields: access code, header, payload.
(Channel) Access Code is used for: synchronization, DC offset compensation and, identification and signaling.
Header contains link control (LC) information: addressing and flow control.

Link control

Standby (device powered on)
Inquiry (discover devices in the area and collect addresses)
Page (connect to a specific device)
Page scan (device waiting for a piconet master)
Connection (actively on a piconet)

Inquiry

Each device wants to know from other devices:

  • device address, used for addressing
  • clock offset, used for synchronization
  • hopping sequence of other devices

Inquiry Packet has unique device address and Channel Access Code.
Receiver Correlator hit causes device to respond with FHS packet containing: device ID and clock(offset). (Frequency hopping scheme)
Inquirer sends multiple inquiries using different frequencies.

Paging

Paging is used to establish a connection. Pager will be master. Other device(s) will be slave.
Master knows slaves device ID and has an idea of its clock.
Master knows hopping sequence and guesses current hopping frequency.

Procedure
Master pages slave (packet has slave ID) at guessed slave page frequency (1 of 32).

  • Master sends page train of 16 most likely frequencies in slave hop sequence.

Slave listens - if correlator triggers, slave wakes-up and relays packet at response frequency. . . (same as inquiry).
Master updates clock information of slave (transmits FHS packet and expects ID packet).

Link Manager Protocol (LMP)

Piconet management

  • attach and detach slaves
  • master-slave switch
  • establishing SCO links
  • handling of low power modes (Sniff, Hold, Park)

Low power mode

Traffic is reduced to periodic sniff slots.
Switch off power after hold offset for hold duration period.
Power saving when keeping more than 7 slaves in a piconet: (Park)

  • Give up active member address, yet maintain synchronization
  • Communication via broadcast LMP messages

L2CAP

Logical Link Control and Adaptation Protocol
Setup and management of baseband connections:

  • Protocol multiplexing
  • Segmentation and re-assembly
  • Quality of service negotiation

Service Discovery Protocol (SDP)

Examples include:

  • Establish L2CAP connection to remote device
  • Query for services
    • search for specific class of service, or browse for services
  • Retrieve attributes that detail how to connect to the service
  • Establish a separate (non-SDP) connection to use the service

Serial port (RS-232) emulation using RFCOMM

Bluetooth 4.0: Low Energy (LE)

Traditional Bluetooth is connection oriented - when a device is connected, a link is maintained.
Sniff modes allow devices to sleep, reducing power consumption to give months of battery life.

Bluetooth Low Energy

Mostly new PHY, some parts derived from the Basic Rate (BR) radio.
New advertising mechanism, for ease of discovery & connection.
Asynchronous connection-less MAC: used for low latency, fast transactions (e.g., 3 ms from start to finish).
Optimized for ultra-low power, enables coin cell battery use cases.
New Generic Attribute Profile to simplify devices and the software that uses them.
For Bluetooth low energy, data throughput is not a meaningful parameter.
It is designed for sending small chunks of data (exposing state). can be used in IoT.
Advertising, Data transactions, Low-latency connection.

Ad-hoc Network

Mobile Ad Hoc Networks (MANET)

  • Are formed by wireless hosts which may be mobile
  • Without (necessarily) using a pre-existing infrastructure
  • Routes between nodes may potentially contain multiple hops

Routing protocols in MANET

  • Proactive protocols
    • determine routes independent of traffic pattern
    • (previously mentioned traditional link-state and distance-vector routing protocols are proactive)
  • Reactive protocols
    • maintain routes only if needed
  • Hybrid protocols

Flooding for data delivery

Data packets are broadcasted to all neighbors, and neighbors further broadcast the data to their neighbors, etc.
Advantages:
Simplicity - no routing tables at all!
Perfect for broadcast.
Potentially higher reliability of data delivery (because packets may be delivered to the destination on multiple paths)
Disadvantages:
Potentially lower reliability of data delivery

  • flooding uses broadcasting – hard to implement reliable broadcast delivery without significantly increasing overhead
  • potentially several overlapping transmissions – high potential for collisions, has to be resolved by MAC, but. . .

Nowadays rarely used in its “pure” form, but a part of numerous algorithms!

Flooding of control packets

Control packets are used to discover routes.
Broadcast storm problem

  • high probability of collisions
  • Redundancy: a given node may receive the same route request from too many nodes
    Solutions to broadcast storm
  • Probabilistic scheme: on receiving a route request for the first time, a node will re-broadcast (forward) the request with probability p
  • re-broadcasts by different nodes should be staggered by using a collision avoidance technique (wait a random delay when channel is idle)
  • Counter-Based Scheme: If node E hears more than k neighbors broadcasting a given route request, before it can itself forward it, then node E will not forward the request
  • Distance-Based Scheme: If node E hears RREQ broadcasted by some node Z within physical distance d, then E will not re-broadcast the request

Reactive routing

Dynamic Source Routing (DSR)

Use separate route request(RREQ)/route reply (RREP) packets to discover route.

  • source node initiates route discovery flooding RREQ
  • each node appends own identifier when forwarding RREQ
  • destination responds with RREP sent on the reversed route

RREP is sent on a route obtained by reversing the route appended to received RREQ.
Data delivery in DSR: Intermediate nodes use the source route included in a packet to determine to whom a packet should be forwarded.

DSR Optimization: route caching

Each node caches a new route it learns by any means.
When node E forwards data [S,E,F,J,D] it learns route [E,F,J,D] to node D.
A node may also learn a route when it overhears data packets.
Use of route caching
Node X on receiving a RREQ for some node D can send a RREP if node X knows a route to node D.
Route error (RERR)
Nodes hearing RERR update their route cache to remove link ERROR.

Route caching

  • Stale caches can adversely affect performance
  • With passage of time and host mobility, cached routes may become invalid

Ad-hoc On demand Distance Vector routing (AODV)

AODV attempts to improve on DSR by maintaining routing tables at the nodes, so that data packets do not have to contain routes.

Discovery

When a node re-broadcasts a RREQ, it sets up a reverse path pointing towards the source.

Route reply

An intermediate node (not the destination) may also send a RREP provided that it knows a more recent path than the one previously known to sender S.
To determine whether the path known to an intermediate node is more recent, destination sequence numbers are used.

Why sequence numbers in AODV?
To avoid using old/broken routes and To prevent formation of loops.
At most one next-hop per destination maintained at each node.

  • multi-path extensions can be designed
  • DSR may maintain several routes for a single destination

Proactive protocols in ad-hoc networks

General idea: take a “classical” protocol, and try do reduce its complexity. . .

Destination-Sequenced Distance Vector (DSDV)

Based on distributed Bellman Ford procedure.
Adds aging information (sequence numbers) to route information propagated by distance vector exchanges; helps to avoid routing loops.
Periodically send full route updates.
On topology change, send incremental route updates.

Associativity-Based Routing (ABR)

Only links that have been stable for some minimum duration are utilized.
motivation: if a link has been stable beyond some minimum threshold, it is likely to be stable for a longer interval.
Prefer paths with high aggregate association stability.

Geographic routing

Alternative: Implicitly infer routing information for next hop from physical placement of nodes:

  • position of current node, current neighbors, destination known - send to a neighbor in the right direction as next hop
  • geographic routing

Options:

  • send to any node in a given area - geocasting
  • use position information to aid in routing - position-based routing (might need a location service to map node ID to node position)

Basics of position-based routing

  1. “Most forward within range r” strategy:
    send to that neighbor that realizes the most forward progress towards
    destination.
  2. Nearest node with (any) forward progress:
    idea: minimize transmission power
  3. Directional routing:
    choose next hop that is angularly closest to destination
    choose next hop that is closest to the connecting line to destination
    problem: Might result in loops!

Mesh networks

A wireless mesh network (WMN) is a communications network made up of radio nodes organized in a mesh topology.
Mesh networks: Multihop, Some nodes mobile, some fixed, Relies on infrastructure, Most traffic is user-to-gateway.

IEEE 802.11s Mesh WLAN

Mesh point (MP): establishes peer links with MP neighbors.
Mesh access point (MAP): MP + AP functionality
Mesh portal (MPP): Gateway.

Station(STA)
Goal: Achieving 802.11 LAN segment behavior.
Through mesh:

  • Broadcast LAN (transparent forwarding)
  • Overhearing of packets (bridge learning)
  • Support for bridge-to-bridge communications (mesh portals being a part of the spanning tree protocol)

Need to determine if the destination is inside or outside the mesh:

  • Inside mesh: Layer2 mesh path discovery/forwarding
  • Outside mesh: which MPP is the right one?
    • If known, unicast to this node. If unknown, send to all MPPs.

IEEE 802.11s routing

Layer2 routing based on MAC addresses.
Hybrid Wireless Mesh Protocol (HWMP) (default routing protocol)
Radio-Aware Optimized Link State Routing (RA-OLSR) (optional routing protocol)

HWMP routing

combines the advantages of on-demand and proactive routing.
On-demand route discovery:

  • Used in intramesh routing for routing optimizations
  • Radio Metric AODV (RMAODV) (Ad-hoc On demand DV)
    Proactive routing:
  • Used when a root portal (MPP) is present
  • Tree based routing
    Main idea:
    Route discovery:
  1. reverse paths set up by broadcast/unicast path requests (PREQ)
  2. path replies (PREP) are sent unicast by destination or the intermediate nodes

HWMP proactive routing

Tree-based routing:

  1. Proactive RREQ mechanism:
  • root issues a broadcast RREQ (route request)
  • MPs may respond with unicast RREP (route reply)
  • 2-way handshaking
  1. Proactive RANN mechanism:
  • root issues a broadcast root announcement (RANN)
  • MP responds with a RREQ
  • root responds with a RREP
  • 3-way handshaking

Radio-Aware OLSR (RA-OLSR)

Optional, Proactive(changes in the link state are advertised to neighbors).
Key concepts:
Multipoint relays (MPRs): cover 2-hop neighborhood, only MPRs send updates(message reduction in space)
Link state reduction (optional): general idea: farther away nodes are updated with less frequency, message reduction in time.

IEEE 802.16j - WMAN Relay/Mesh

Wireless Metropolitan Area Network
Scope:

  • Support OFDM PHY mode
  • MAC enhancements
  • Enable different base-station operations: MMRBS (Mobile Multihop Relay) and RS (Relay Station)

Extend coverage:

  • Coverage black holes can be covered by relays
  • A series of relays can forward the traffic to the base station

Load balancing: Divert traffic from overloaded base stations.

IEEE 802.15.5 - WPAN Mesh

Scope:

  • Routing redundancy and coverage extension
  • Both low-rate and high-rate WPAN:
    • control and maintenance networks
    • wireless streaming applications
      Examples: Home automation & Industrial control and automation.
