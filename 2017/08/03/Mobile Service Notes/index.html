Mobile Service Notes

Posted on 2017-08-03

Intro

The Emergence of Mobile Services Web 2.0 Term that represents different concepts and paradigms of latest WWW services:

  • “The web as a platform” – Software applications are based on the web, not the PC
  • Democratization – Community service and user-generated content
  • Mashups – new services through combination of existing services and content

Cloud Computing

  • Infrastructure as a Service: virtualization of computer hardware
  • Platform as a Service: usage of service platforms without the cost and complexity of buying and managing underlying hardware and software layers
  • Software as a Service: loading software into the user’s web browser on demand and executing it there; related data is stored in the cloud

Types of mobile networks

  • Cellular networks
  • Ad-hoc networks
  • Mesh networks
    • Special category of an ad-hoc network where (mobile) nodes are organized in a mesh topology
    • Mesh client: laptop, cell phone, or another other wireless device
    • Mesh router: mobile and fixed node that forward and route traffic between clients, other routers, and gateways over wireless links
    • Gateway: component that provides interconnectivity between mesh network and an infrastructure-based network

Pervasive Computing

  • “Everywhere, everyone, at any time” paradigm
  • Takes into account the environment in which information and communication technology is used, for example, toys, computers, cars, homes, factories, work areas
  • Based on integrated processors, sensors, and actuators connected through high-speed networks and combined with new devices for viewing and display

Ubiquitous Computing

  • Interconnecting computing devices with environmental objects
  • Integration of computers into practically all objects in our everyday environment, endowing them with computing abilities

What is a Mobile Service?

  • Remotely accessible at well-defined interfaces and offered over a mobile communications system
  • Invoked and used by protocols of the application layer and possibly supported by enabling functions of the underlying mobile network
  • Follows one or several of the paradigms of mobile computing, ubiquitous and pervasive computing, and location and context-awareness and combines them with a business model

Mobility Classification

  1. Terminal Mobility
    • Refers to the seamless mobility of mobile devices
    • Registration, call, and connection between mobile device and network are kept while in motion
    • Key functions: wireless communications, handover, location management, roaming
  2. Personal Mobility
    • Means that users can access their services according to their subscription from different devices and device types
    • User profiles and services are available across device boundaries
    • Key functions: authentication mechanism
  3. Service Mobility
    • Enables usage of tailored and personalized services even if the user is roaming to foreign networks
  4. Session Mobility
    • Allows to interrupt sessions and to resume them later possibly from another terminal or another network

Physical layer
Tasks: frequency selection, generation of carrier frequency, signal detection, modulation of data onto a carrier frequency, and encryption

Data link layer
Medium access, multiplexing, detection and correction of transmission errors, and detection of data frames (synchronization)

Mobile networks

Infrastructure-based Networks

Access points/base stations serve limited geographic areas, typically known as radio cells.
Complexity resides at the infrastructure site, whereas the terminal can remain comparatively simple.

Important procedures in infrastrcuture-based networks:

  • Handover: changing the assignment of a mobile terminal from one access point/base station to another as the mobile moves from one radio cell to another.
  • Paging: procedure initialized by the infrastructure to determine the access point/base station a certain terminal is attached to.
  • Location update: procedure initialized by a mobile terminal to notify the network about the current access point/base station it is connected to.
  • Roaming: use of services in a foreign network, i.e., the network of a mobile network operator the subscriber has not contract with.

Infrastructure Components

  • Access points
    • Medium access control in the uplink
    • Execution of handover and paging
    • Power Control
  • Router and Switches
  • Databases
    • Localization (e.g., association between user and terminal and between device and access point)
    • Storage of user profiles and user data
    • Data for authentication
  • Server
    • Support for application services (i.e., services above connectivity services)

GSM (Global System for Mobile Communications)
GPRS (General Packet Radio Service)

  • Extension of GSM for packet-switched data transmission
  • Introduction of a second, packet-switched core network, new software releases for existing access networks
    EDGE (Enhanced Data Rates for GSM Evolution)

GSM Network Architecture

Base Station Controller, Transcoding and Rate Adaption Unit, Mobile Switching Center, Visitor Location Register, Gateway MSC, Public Switched Telephone Network, Integrated Services Digital Network, Equipment identity Register, Authentication Center.

SIM (Subscriber Identity Module)
Provides separation of personal from terminal mobility (e.g., enabling international roaming independent of user equipment and network technology).
Contains cryptographic algorithms to support authentication and user data encryption.
SIM Application Toolkit (SAT): execution environment for operator specific functions.
Base Station Subsystem (BSS): Consists of a BSC and one or more BTSs.

BTS

  • Each BTS is allocated a set of frequencies (Cell Allocation, CA)
  • In order to reduce complexity, signal and protocol processing is limited to error protection, encryption, and link level signaling.

BSC: Performs essential control functions and coordination between BTSs, e.g.

  • Reservation of radio frequencies
  • Management of handover from one cell to another within the same BSS
  • Control of paging
  • Active power control
  • Set up of signaling channels

TRAU

  • Reduce standardized SS7 PCM stream from 64kbits/s to 13kbits/s for the air interface
  • Usually build into the MSC to save costs on the link between MSC and access network

MSC

  • Difference between conventional fixed network switch and MSC: allocation and administration of radio resources and mobility of subscribers (supports location registration, handover between different BSCs, …)
  • Support of service features like call forwarding or conference calls
  • Signaling between MSCs and between MSCs and other logical entities accomplished by the Signal System No. 7
  • operating in circuit-switched mode cannot be used for packet-switching

Gateway MSC

  • Passing of voice traffic between fixed and mobile networks
  • Required as access to GSM network, because fixed network is unable to connect an incoming call to the local target MSC (due to its inability to interrogate the HLR)

VLR

  • Database containing distributed nodes, each being responsible for a certain coverage area
  • Contains information about subscribers currently physically staying in the associated coverage area
  • Usually combined with an MSC

EIR

  • Stores all user equipment identification registered for this network
  • Maybe used for detecting stolen devices

GPRS(General Packet Radio Service)

PCU (Packet Control Unit)
SGSN (Serving GRPS Support Node)

  • Connects the radio access network to the GPRS/UMTS core
  • Tunnels user sessions to the GGSN
  • Packet-switched counterpart to MSC
    GGSN (Gateway GPRS Support Node)
  • Gateway that provides mobile subscribers access to the Internet
  • Counterpart to the circuit-switched GMSC

UMTS

Universal Mobile Telecommunications System

  • Medium access: CDMA Direct Sequence with a channel bandwidth of 5 MHz and 3.84 Mbps chipping rate
  • FDD mode: 1920-1980 MHz for uplink and 2110-2170 for downlink
  • TDD mode: 1900-1920 MHz and 2010-2025 MHz

HSPA (High-Speed Packet Access)
HSDPA (High-Speed Downlink Packet Access)

  • Advanced modulation schemes (16QAM and 64 QAM) for UMTS networks

Node B (NB)

  • Significant differences to BTSs regarding medium access (CDMA instead of FDMA/TDMA), power control, etc
  • UE is usually connected to several NBs simultaneously

RNC (Radio Network Controller)

  • Counterpart to GSM’s BSC
  • Neighboring RNCs are directly interconnected (higher complexity, but better handover control)

LTE (Long Term Evolution)

  • Introduces multicarrier modulation (OFDM) and new antenna technology (MIMO)
  • IP-based protocols on all interfaces
  • Split in the control/user plane between MME and S-GW/PDN-GW

WiMAX (Worldwide Interoperability for Microwave Access)

  • Specification for a 4G network that delivers high-speed broadband, fixed and mobile services wirelessly to large areas with much less infrastructure using IEEE 802.16 standard
  • Used of multicarrier modulation (OFDM) and Multiple-Input-Multiple-Output (MIMO) antennas (similar to LTE)

PDN-GW (Packet Data Network Gateway).

S-GW (Serving Gateway)

  • Routes and forwards user data packets
  • Acts as mobility anchor for the user plane during inter-eNB handover and as anchor for mobility between LTE and other 3GPP technologies
  • Terminates downlink data path for idle UEs and triggers paging

MME (Mobility Management Entity)

  • Tracking of idle UEs
  • Paging procedure and retransmission
  • Bearer activation and deactivation
  • Handover control
  • Authentication

WiMAX
Deployment Scenarios

  • Connecting WLAN access points to the Internet
  • Providing a wireless alternative to cable and DSL for “last mile” broadband access
  • Providing data and telecommunications services
  • Providing portable connectivity
  • Enabling large range mesh networks

WiMAX bridges the gap between WLAN (high data rates, but low mobility support) and cellular networks (low data rates, but high mobility support).
IEEE WLAN 802.11
Set of standards for wireless area networks specified by IEEE
Does not fix an entire network infrastructure with high-level services (like GSM), but is only an access technology.

Bluetooth
Single-chip, low-cost, radio-based wireless network technology for connecting different small devices and/or peripherals in an ad-hoc manner, i.e., without expensive wiring or the need for a wireless infrastructure.
Coverage range limited to 10m typically.

Piconet

  • Basic unit of networking in Bluetooth
  • Consists of a master (M) and up to seven slaves (S)
  • Slaves only communicate with the master and only when granted permission by the master

Scatternet (Group of linked piconets joined by common devices)

Mobility Management

Handover

Handover Management

Maintaining the traffic connection with a moving user when crossing cell boundaries.
Occurs when the quality or the strength of the radio signal falls below certain parameters (signal quality reason)
Occurs when the traffic capacity of a cell has reached its maximum or is approaching (traffic reason)
GSM standard identifies about 40 reasons for a handover.

Location Management

Mechanisms to localize users in case of incoming calls, short messages, or data.
Requires to partition an operator’s coverage area into location areas in order to efficiently perform location.
Two basic operations: Location update and Paging.
Location Update (LU): operation initialized by the terminal to inform the network about the user’s location.
Paging: broadcast message initialized by the network to locate the current cell of a user.
Measurement -> Decision -> Execution.
Measurement: Received signal strength(taken as example)

Decision: RSS with Threshold and Hysteresis

Handover occurs only if

  • the current signal level drops below a threshold, and
  • the target base station is stronger than the current one by a hysteresis margin H
    Scheme avoids the ping-pong effect and execution of handover if signal from the serving base station is still strong enough.

Hard handover

“Break before make”
Causes a short cut in the connection
Primarily used in FDMA and TDMA, where different frequency ranges are used in adjacent cells.

Soft handover

“Make before break”
Used in CDMA, where adjacent cells use the same frequency range.

Handover Control

Network-controlled Handover (NCHO)

  • Mobile terminal makes no measurements
  • No handover signaling at the air interface
  • Takes up to 10 seconds

Mobile-assisted Handover (MAHO)

  • Mobile terminal continuously measures signal strength from serving and neighboring base stations and sends the recorded values to the serving base station
  • the network decides when handover should take place

Mobile-controlled Handover (MCHO)

  • Mobile terminal decides on handover
  • Very short reaction time (on the order of 0.1 seconds)

Location Areas

Several cells are combined to a location area (LA).
When the system must establish a communication with the mobile, the paging only occurs in the current LA where called user resides.
Location information are stored in databases (generally, a home database and several visitor location databases are included in the network architecture)
Goal: minimizing location management cost (LU+paging traffic and processing)

Dynamic Location Update Schemes

Selective LA Update
Location update process at certain LAs is skipped, as the user might spend a very short period of time in those LAs.
Time-based
User updates his location at constant time intervals.
Profile-based
Network maintains a profile for each user.
Profile consists of a sequential list of the most likely LAs where the user is located at different time periods.
LAs on the list are paged sequentially from the most to the least likely LA.
Profile should be updated from time to time.
Movement-based
User updates its location after a given number of boundary crossings to other cells in the network.
Distance-based
User updates his location when it has moved away a certain distance from the cell where he has last reported his location.

Paging Strategies

Blanket Polling/Simultaneous Paging

  • All cells in the user‘s LA are paged simultaneously
    Shortest-distance-first
    Network pages the user by starting from the last cell where the user has updated his location.
    Sequential paging
    User is paged sequentially in sub-groups of cells in the LA
    Sub-groups are ordered according to their estimated probabilities of having the user located in them.
    Velocity paging
    Users are classified based on their velocities at the moment of location updates.
    Paging area is then dynamically generated based on the user‘s location update time and velocity class index.

GSM Location Management

GSM circuit-switched: Hybrid location updating
GPRS: introduction of new (smaller) location areas with adaptive paging/location update strategies.
Hybrid location updating

  • Time-based
    • Executed by the terminal if a timer has expired
    • Timer value is determined by operator
  • Forming LAs
    • Mobile station recognizes new location area by reading the Location-Area-Identifier broadcast
    • If new location area is recognized, location update is triggered

Numbering Schemes for PSTN/ISDN

Country Code(CC) 1-3 digits
National Destination Code (NDC) N digits
Subscriber Number (SN) 15-N-CC digits

Basics of Numbering in GSM

Permanent numbering

  • Numbering of subscribers
    • IMSI (International Mobile Subscriber Identity)
      Uniquely identifies the subscriber and is stored in the SIM, HLR, and AuC.
    • Mobile Subscriber ISDN Number (MSISDN)
      Real telephone number of a subscriber.
  • Numbering of devices
    • IMEI (International Mobile Station Equipment Identity)

Temporary numbering

  • MSRN (Mobile Station Roaming Number)
    Temporary location-dependent ISDN number. Required to make routing decisions and to identify the responsible MSC. Assigned by the locally responsible VLR to each mobile station in its area and passed to the HLR.

  • TMSI (Temporary Mobile Subscriber Identity)
    Assigned during the mobile station’s presence in the area of one VLR (by that VLR) and is usually changed during this period (ID hopping).
    Subscriber can be uniquely identified; IMSI is replaced by (TMSI, LAI).

  • LAI (Location Area Identifier) Regularly broadcasted by the base station
  • CI (Cell Id)

Delivery of a Mobile-Terminated Call

PSTN-> GMSC(HLR) -> MSC(VLR)-> BSC -> BTS (Paging) -> UE
MSISDN -> MSRN -> LAI + TMSI -> Paging TMSI in LA

Location Registration

Must be executed in order to get access to a GSM network, i.e., to receive or to initiate calls.
Steps: 1) Sending IMSI and LAI to the network. 2) Authentication. 3) Start of ciphering. 4) Generation of an TMSI (stored in the associated VLR) and an MSRN (stored in the central HLR). 5) MS receives TMSI and saves it in the SIM storage.
Difference between location registration and location update: location update is based on TMSI.

Location Update

UE -> BTS ->BSC -> new MSC(VLR) ->Old MSC(VLR) and HLR
HLR -> Old MSC(VLR)
new MSC(VLR) -> BSC -> BTS -> UE
UE update Location (contains old TMSI as well as old and new LAI).

New VLR

  • request parameters from old VLR
  • Update location in HLR
  • Location update confirmation and delivery of a new TMSI
    HLR notifies old VLR to remove parameters.

Number Portability

MSISDNs are operator-specific and initially could not be kept by the user when changing the operator (number portability).
Number Portability Database (NPDB) : operational database (used in real time at call set-up) which provides portability information.

GPRS Location Management

Introduction of Routing areas and State model for adaptive location management.
Routing Areas
A routing area comprises several cells.
Routing areas are significantly smaller than location areas.
Timers: READY, STANDBY
State transitions are executed by timers, data packet transfer, or user activity.

UMTS Location Management

New approach for UMTS PS domain (Packet switched)

  • Track subscribers on the basis of routing areas in the core network
  • Track subscribers on the basis of UTRAN Registration Areas (URAs) and cells in the access networks
    PMM (Packet Mobility Management)

LTE Location Management

LTE re-introduced the single LA structure of GSM and distinguishes between three UE states.
New name for location areas: tracking areas (TAs).
MME allocates a list with one or more TAs to the user.
In certain modes of operation, the UE may move freely in all TAs of the list without updating the MME.
New feature: the arrangement of TAs may be dynamically changed in a self-organizing manner based on observed mobility patterns.

UE states:

  • LTE-Active
    Network knows the cell the UE belongs to, and UE can transmit and receive data to and from the network.
  • LTE-Idle
    Network knows the location of the UE at the granularity of TAs.
    UE is in power-saving mode and sends a TA only when crossing TA borders.
  • LTE-Detached
    Network has no information about the current cell or TA of the user – no packet transfer possible.

Mobile Internet

Problem: TCP connection cannot survive IP address change.

Mobile IP

Foreign Agent

  • Implemented in the router of the foreign network
  • Acts a tunnel endpoint that decapsulates packets and forwards them to the MN

Care-of Address (CoA)
IP packets sent to the MN are delivered to the CoA, not directly to the MN‘s IP address.
CoA marks the tunnel endpoint.

Basic Procedure

1) CN-> HA
2) HA-> FA. HA encapsulates the datagram inside a new IP datagram with the MN’s CoA in its header.
3) FA-> MN
4) MN-> CN

Agent Advertisement

HAs and FAs periodically broadcast their presence using an Agent Advertisement Message (extension of ICMP messages).
IP-in-IP Encapsulation
Mandatory of Mobile IP
Drawback: several redundant fields
Minimal Encapsulation
Removes redundant fields
Source address in the inner header is omitted if original sender is identical to the HA.

Optimization

Idea: Optimize routing by informing the CN of the current location of the MN.
1) Binding request (CN->HA)
Any CN that wants to know the current location of an MN sends a binding request to the HA.
If HA is allowed to disclose the MN’s location, it returns a binding update
2) Binding update (HA->CN)
3) Binding acknowledgement (CN->HA)
4) Binding warning
If a FA decapsulates a packet for a MN, but is not the MN’s current FA, it sends a warning to the CN.
Warning contains the MN’s fixed home address.
After receiving a warning, the CN knows that the MN has probably moved, and initiates a binding request.

Mobile IPv6

Main differences compared to Mobile IPv4:
▪ No need for special Foreign Agent routers.
▪ Route optimization is part of the protocol, not an extension.
▪ Most packets sent to a mobile node while away are sent using an IPv6 routing header instead of IP encapsulation.

Network Layer Support in GPRS/UMTS

Basic Procedure
Before data can be transmitted between the UE and an external host, some preparations are necessary:

  • Appropriate SGSN must be found (according to the user’s location)
  • Features and capabilities of the data service must be negotiated
  • An GGSN must be selected and a setup between UE and GGSN has to be established
  • Path for routing data packets has to prepared for tunneling

Three procedures:

  • GPRS Attach
  • Activation of PDP Context (Packet Data Protocol)
  • Data Transfer

Several PDP contexts can be set up for a UE such that the UE can get connected to several IP networks with different QoS.

PDP Context

Contains routing information for packet transfer between a UE and a GGSN to have access to an external packet-switching network.
Stored in the mobile station, the SGSN, and the GGSN.
visible for the external network.
Access Point Name (APN): Logical name for the desired data network.
PDP address: Address assigned to the mobile station (e.g. an IP address).

Packet TMSI (P-TMSI)

  • Assigned during an GPRS attach procedure and after a location update.
  • Used to page the UE when packets have to be delivered
  • Mapping between PDP address and P-TMSI by the SGSN makes the transmission of packets between GGSN and UE possible

Data Transfer

Gateway Tunneling Protocol (GTP) between GGSN (Home Agent) and SGSN (Foreign Agent).

SIP

SIP is an application layer protocol that is used for establishing, modifying, and terminating multimedia sessions in an IP network.
Main signaling functions of the protocols:

  • Location of an end point
  • Contacting an end point to determine willingness to establish a session
  • Exchange of media information to allow a session to be established
  • Modification of existing media sessions
  • Teardown of existing media sessions

Design principles
Separation of signaling and media description – can add new applications or media.
Extensibility

Components

User Agent Client
Caller application that initiates requests
User Agent Server
Accepts, redirects, rejects requests and sends responses to incoming requests on behalf of the user.
Proxy server
Receives and forwards SIP requests
Core component for mobility support.
Registrar

URI

A SIP address is not tied to a single physical device but instead is a logical entity that may change its location, SIP uses the term URI instead of URL.

User URI

Also known as Address of Record.
Request sent to a User URI requires database lookups and service and feature operations, which can result in the request being sent to one or more devices.
Used in the To and From fields of various SIP headers.

Device URI

Known as a Contact
Associated with a user for a shorter period of time.

SIP URIs

Follows the scheme
sip:userinfo@host:port[parameters][headers]
Examples:

Tel URI

Used to identify resources using a telephone number according to E.164.

Establishing a Session in a P2P Fashion

Type of message; Request-URI to which the message is sent.
Each host that originates or forwards a message stamps its own address in the Via header field, written as host name that can be resolved into an IP address by using DNS.
Note: Message is routed to the Request-URI and not to address in the To field, because Request-URI can be changed and rewritten.
Tag in the From field is generated by the session initiator.
Tag in the To field is generated by the user agent answering the request.
Contact: Device URI of communication device used by the initiator.

received parameter contains the IP address that the request was received from (which typically is the same address that the URI in the Via resolves using DNS)
To and From indicate the direction of the request, not the direction of the message.
Response also contains a Contact field, which delivers the address at which Marconi can be contacted directly once the session is established.

SIP Proxy Server
Intermediate node between devices that supports session setup.

Mobility Support

Personal Mobility
Personal mobility is achieved by using the REGISTER method, which allows a mobile device to change its IP address and point of connection to the Internet and still be able to receive incoming calls.

Terminal Mobility

  • Mobile IP
  • Built-in mobility support at the application layer

Service mobility means that the same set of proxies are used to route incoming and outgoing requests when mobile.
Precall Mobility
Midcall Mobility

Mobile device detects a new wireless network and uses DHCP to acquire a new IP address.
Device then performs a re-INVITE to make the signaling establish a signaling and media flow to the new IP address.

IMS

IP Multimedia Subsystem
Enriched Services over Packet Networks

Session

Connection between two or more communication end points (e.g., mobile phones, PDAs, …)
IMS enables any type of media sessions to be established (e.g., voice, video, text) and for sessions to be dynamically modified on the fly.
IMS adopts the Session Initiation Protocol (SIP) for session control.

  • Multiple Services - Single Session
    User can alter media types within a single session
    Converged services which bundle together multiple service components
  • Synchronized Services - Multiple Sessions
    Interlinked services which can trigger other services in independent sessions.
  • Unrelated Services - Multiple Sessions
  • Number of unrelated services running in parallel, independent sessions.

Architecture

CSCF (Call Session Control Function)

  • CSCFs are SIP Proxies
  • Does not route user data
  • Three different version: P-CSCF, I-CSCF, S-CSCF
  • Performs signaling operations for SIP session control (e.g., establishing, maintaining, and terminating sessions)
  • Coordinates with other network entities for session and service control and resource allocation

MRF (Media Resource Function)

  • Processing and generation of multimedia content
  • Real-time transcoding of multimedia data (i.e., conversion between different formats)

MGCF (MGW Control Function)
MGW (Media Gateway Function)

  • Interconnects circuit switched channels and media streams from packet networks
  • Media conversion
  • Bearer Control

Proxy CSCF

  • First contact point for users within the IMS

Interrogating CSCF

  • Contact point within an operator’s network for all connections destined to a subscriber of that network.
  • Obtains the name of the next hop from the HSS
  • Assigns an S-CSCF based on received capabilities from the HSS.

Serving CSCF

  • Focal point of IMS: handles registration processes, makes routing decisions, maintains session states, and stores service profiles.

LBS

Location based services

GPS

Positioning Infrastructure

  • Terminal attached to the target person
  • Reference stations at well-known coordinates
  • Control units for coordinating positioning

Satellite constellation: number and positions of satellites in space.
Constellation has been arranged in a way that any point on the Earth’s surface is covered by at least four satellites.
Ephemeris
Contains all information needed by the receiver to compute the exact satellite position in space when performing measurements.

Almanac
Subset of each satellite’s ephemeris and clock data.

Positioning Step 1 - Identification of Satellites

GPS receiver must in the first step determine the satellites to be used for range measurements.
5 to 10 satellites are typically in the visible horizon of a receiver.
Cold start-up (40-60s)

  • Receiver has no information about its last position and no valid almanac
  • Receiver must listen to the transmission on the L1 carrier and auto-correlate the received C/A codes with the entire C/A code space

Warm start-up (30-40s)

  • Receiver knows its last position and has almanac
  • Receiver can roughly estimate current satellite constellation

Hot start-up (5-15s)

  • Receiver has valid ephemeris, which allows to determine current satellite constellation very accurately

Positioning Step 2 - Ranging

Code phase ranging for SPS(Standard Positioning Service) and PPS(Precise Positioning Service).
Dual-frequency ranging: PPS uses code phase ranging on both carriers and can thus adjust much better to ionospheric refraction.
Carrier phase ranging can be applied optionally, but is very time consuming and thus not applicable for real-time application like LBSs.
Range measurements must be made to at least four satellites for applying circular lateration in 3D.

Positioning Step 3 - Position Calculation

Time offset between receiver time and GPS time is modeled as an additional unknown in the system of equations for circular lateration.

Differential GPS

Principle
Observe and measure pilot signals from the satellites at a well-known position (reference station) and determine the difference between the measured range and an approximation of the true range derived from the known position of observation.
Derived difference then serves as a correction value for GPS receivers staying close by.

LCS

Overview of LCS Architecture
LCS (Location Service): GSM/UMTS term for positioning method.
Mobile Location Center (MLC): control and coordination of positioning.
Serving MLC (SMLC): control of positioning in the access network.
Gateway MLC (GMLC): interconnectivity to external LCS clients.
Location Measurement Unit (LMU): Timing synchronization of base stations.

Assisted GPS

Motivation

  • Reduce GPS sensor startup time
  • Increase accuracy by DGPS
  • Consume less handheld power than conventional GPS

Mobile-assisted A-GPS

Position calculation is done in the network
Network transmits short assistance data to the terminal.
Terminal performs measurements and returns results to the network for position estimation.

Mobile-based A-GPS

Position calculation is done by the terminal.
Network transmits extended assistance data to the terminal.

Device-centric Positioning

Modern smartphones use a combination of GPS, Cell-Id and WiFi positioning.
Positioning process is supported by a location provider.

Proactive Location-Based Services

Reliability, Communication, Energy, Scalability.
Key Parameters: Sampling Rate (Hover time), Users, Geofences, Environment(Wireless networks, Transport networks, Events).
